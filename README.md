# name_to_love
📌 한글 이름 궁합 계산 프로그램 전체 정리

이번 프로젝트는 두 사람의 한글 이름을 입력받아 각 글자를 숫자로 변환한 뒤, 이를 이용해 궁합 퍼센트를 계산하는 프로그램을 구현한 것이다. 전체 구조는 문자열 처리, 유니코드 변환, 배열 연산, 반복 축소 알고리즘으로 구성되어 있다.

1. 한글을 숫자로 변환하는 과정
입력된 이름은 UTF-8 문자열로 저장된다. 한글은 3바이트로 구성되어 있기 때문에 문자열을 3바이트 단위로 읽어 한 글자씩 분리한다.

이후 각 문자는 유니코드 값으로 변환되고, 한글 범위(0xAC00 ~ 0xD7A3)에 해당하는 경우에만 처리한다. 이를 통해 영어와 특수문자는 제외하고 한글만 선택적으로 추출한다.

2. 초성·중성·종성 분해 및 획수 계산
한글 한 글자는 초성, 중성, 종성으로 구성되어 있으며 유니코드 규칙을 이용해 이를 분해한다.

기준값 0xAC00을 이용해 한글 위치를 계산한 뒤 588 단위 구조를 활용하여 초성, 중성, 종성 인덱스를 구한다. 각 인덱스는 미리 정의된 배열을 통해 획수로 변환된다.

이 과정을 통해 한 글자는 하나의 숫자로 축약된다.

3. 이름을 숫자 배열로 변환 (a, b)
첫 번째 사람의 이름은 배열 a, 두 번째 사람의 이름은 배열 b로 변환된다.

각 배열에는 이름의 각 글자가 숫자로 변환된 값이 순서대로 저장된다.

- a = 첫 번째 사람 이름의 숫자 배열
- b = 두 번째 사람 이름의 숫자 배열

4. 배열 병합 (love 함수 핵심)
두 배열 a와 b는 번갈아가며 하나의 배열 v로 합쳐진다.

a[0], b[0], a[1], b[1] … 순서로 저장되며 두 사람의 정보가 하나의 수열로 결합된다.

5. 반복 축소 알고리즘
병합된 배열 v는 인접한 값들을 더한 뒤 10으로 나눈 나머지를 적용하는 방식으로 반복적으로 줄어든다.

v[i] = (v[i] + v[i+1]) % 10

이 과정을 배열 길이가 2가 될 때까지 반복한다. 이를 통해 숫자는 점점 압축된다.

6. 궁합 퍼센트 생성
최종적으로 남은 두 숫자는 다음과 같이 결합된다.

p = v[0] * 10 + v[1]

이 값이 두 사람의 궁합 퍼센트가 된다.

7. 결과 출력 (result 기능)
계산된 퍼센트 값은 막대 그래프로 시각화된다.

- 전체 20칸 기준 그래프 출력
- ■는 높은 비율, ·는 낮은 비율을 의미
- 퍼센트 숫자 함께 출력

또한 조건에 따라 결과 메시지가 출력된다.

- 90% 이상: 천생연분
- 70% 이상: 잘 어울리는 관계
- 40% 이상: 가능성 있음
- 그 외: 좋은 친구

특정 값(99, 67)에 대해서는 예외 메시지가 출력된다.

8. 전체 동작 흐름 요약
1) 이름 입력  
2) 한글 문자 추출  
3) 획수 기반 숫자 변환  
4) 배열 a, b 생성  
5) 번갈아 배열 v 생성  
6) 반복 축소  
7) 최종 2자리 숫자 생성  
8) 퍼센트 및 결과 출력  

📌 최종 정리
이 프로그램은 문자열을 숫자로 변환하고 반복적인 연산을 통해 하나의 값으로 수렴시키는 구조를 가진 알고리즘이다.

핵심 개념:
- UTF-8 문자열 처리
- 유니코드 기반 한글 분해
- 배열 연산
- 반복 축소 알고리즘
- 결정론적 결과 생성

즉 같은 이름을 입력하면 항상 같은 결과가 나오는 규칙 기반 계산 시스템이다.
